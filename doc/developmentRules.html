# node版本

首先需要安装node，建议选择使用nvm进行安装node，通过nvm来统一托管我们的node版本
在使用的时候只需要进行灵活调配node版本。

TIP

nvm安装教程可以参考如下教程：nvm安装教程
如何使用nvm：nvm官网

  • 当按照以上步骤进行安装完毕后，全局命令行运行nvm --version，提示版本号信息即为安装成功。
  • 执行nvm install 10.15.0进行node@10.15.0的版本安装。
  • 以上操作在 nvm安装教程 中全部涉及。
  • 当node版本安装后记得全局设置一个node版本，全局命令行运行node -v，提示版本号信息即为全局node版本安装成功。

# 新项目初始化

WARNING

以下所有步骤演示均使用vscode，所有人统一开发工具进行开发

经过nvm以及node的安装后，开始初始化我们的项目：我们的项目是基于vue-cli的一个自定义preset进行初始化的，请保证全局或者当前目录下安装了vue-cli。

WARNING

vue-cli的版本为3.3.0，当前3的大版本号都是可以进行初始化，4以上的版本没有经过测试！

TIP

全局安装以及vue-cli文档教程：vue-cli官网，具体问题可以进行查看

# 全局安装vue-cli

npm install -g @vue/cli
# OR
yarn global add @vue/cli

安装成功后全局命令行运行vue --version，提示版本号即为安装成功
新项目初始化下执行，即可在当前目录下新建我们预设好的preset

vue create --preset wonderfulMan/counselor-preset-mobile . // 当前目录创建
vue create --preset wonderfulMan/counselor-preset-mobile test-module // 当前目录创建test-module

# 局部安装vue-cli

我们推荐采用项目内安装来进行vue-cli的初始化，这样可以保证不同的版本不受全局影响
新建test文件夹，在当前目录打开命令行运行

npx vue@3.3.0 create --preset wonderfulMan/counselor-preset-mobile . // 当前目录创建
vue create --preset wonderfulMan/counselor-preset-mobile test-module // 当前目录创建test-module

# 初始化步骤

TIP

在执行以上初始化命令时，请确保当前node版本为10以上

以下已当前目录创建

选择y把当前目录作为工作目录，选择n退出初始化，等待vue-cli初始化项目，当 显示以下信息，则表示初始化成功！

需要注意vscode编辑器是否禁用了vscode，如果出现以下标志请点击图中红色文字，根据弹窗提示开启eslint功能

在当前工作根目录新建.env文件和.env.development.local文件

.env文件内容如下

  VUE_APP_DEV_SERVER_MODULE=模块名称

.env.development.loacl文件内容如下

VUE_APP_DEV_SERVER_PORT=本地服务端口（例如：80）
VUE_APP_DEV_SERVER_TARGET=接口代理地址（例如：http://172.31.16.50:8082）
VUE_APP_DEV_SERVER_MODULE=模块名称

当前工作目录命令行运行

yarn serve

看到命令行如下显示，项目启动成功

接下来需要进行下载counselor-mobile-ui和counselor-sdk进行本地link操作 （请联系前端同事获取权限，不然无法获取源码）

首先从git上获取counselor-mobile-ui和counselor-sdk源代码后，vscode打开分别打开项目
在counselor-mobile-ui 和 counselor-sdk的根目录yarn link后，在当前项目内执行分别执行

yarn link counselor-sdk-ui 
yarn link counselor-sdk

执行以上全部操作后一个新项目就可以开发了！
